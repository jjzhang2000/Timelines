# 需求规格说明书：Timelines - 通用历史时间线框架

## 1. 执行摘要

Timelines 是一个基于 Flutter 的通用时间线应用程序框架，支持通过加载 JSON 数据文件展示任意主题的历史时间线。该应用提供多数据源合并显示、对比视图、搜索和文章系统等核心功能，让用户能够灵活地探索和分析不同领域的历史事件。

## 2. 项目概述

### 2.1 目的
- 提供通用的时间线展示框架，不限定特定主题
- 支持通过 JSON 文件驱动内容，实现数据与展示分离
- 支持多数据源同时加载，提供合并视图和对比视图
- 提供直观的导航、搜索和筛选功能
- 为每个事件支持详细的文章展示

### 2.2 目标用户
- 历史研究者和教育工作者
- 需要对比分析多个时间线的专业人士
- 希望自定义时间线内容的爱好者
- 需要可视化展示历史进程的用户

### 2.3 平台支持
- 移动平台（iOS、Android）
- 网页浏览器
- 桌面应用程序（Windows、macOS、Linux）

## 3. 功能需求

### 3.1 核心时间轴功能

#### 3.1.1 时间轴显示
- **REQ-TL-001**: 应用程序应显示历史事件的垂直时间轴
- **REQ-TL-002**: 事件应按时间顺序从顶部（早期）到底部（近期）排列
- **REQ-TL-003**: 时间轴应支持缩放和平移以进行详细导航
- **REQ-TL-004**: 应用程序应为每个事件渲染视觉元素（线条、气泡、标签）
- **REQ-TL-005**: 不同的事件类型（时代与事件）应有视觉区分
- **REQ-TL-006**: 来自不同 JSON 文件的事件应使用颜色/标签区分来源

#### 3.1.2 导航
- **REQ-NAV-001**: 用户应能够垂直滚动浏览时间轴
- **REQ-NAV-002**: 应用程序应支持基于手势的导航（滚动、捏合缩放）
- **REQ-NAV-003**: 用户应能够跳转到特定时间段或事件
- **REQ-NAV-004**: 应用程序应在导航期间保持上下文（当前时代高亮）
- **REQ-NAV-005**: 滚动和缩放期间应提供平滑动画

### 3.2 数据源自动发现

#### 3.2.1 数据源加载
- **REQ-SRC-001**: 应用程序应从固定文件夹自动扫描并加载所有 JSON 数据文件
- **REQ-SRC-002**: 应用程序应支持通过配置文件指定数据源列表
- **REQ-SRC-003**: 应用程序启动时应自动发现并加载所有可用的数据源
- **REQ-SRC-004**: 每个数据源应有独立的配置（名称、颜色方案）

#### 3.2.2 显示模式
- **REQ-MODE-001**: 应用程序应支持合并视图模式（所有启用的数据源事件混合显示）
- **REQ-MODE-002**: 应用程序应支持对比视图模式（多个时间线并排显示）
- **REQ-MODE-003**: 用户应能够在设置中切换显示模式
- **REQ-MODE-004**: 合并视图中，不同来源的事件应使用颜色编码区分
- **REQ-MODE-005**: 对比视图中，每个数据源应有独立的垂直时间轴

### 3.3 搜索系统

#### 3.3.1 搜索功能
- **REQ-SEARCH-001**: 用户应能够按名称或关键词搜索事件
- **REQ-SEARCH-002**: 搜索应支持前缀匹配以高效查找
- **REQ-SEARCH-003**: 搜索结果应按时间顺序显示
- **REQ-SEARCH-004**: 应用程序应在输入期间提供自动补全建议
- **REQ-SEARCH-005**: 搜索应跨所有事件标签和描述执行
- **REQ-SEARCH-006**: 搜索算法应支持基于单词的分词和驼峰命名识别
- **REQ-SEARCH-007**: 多词搜索查询应在术语之间使用 AND 操作
- **REQ-SEARCH-008**: 搜索应支持大小写无关匹配，同时保留原始显示格式
- **REQ-SEARCH-009**: 搜索结果应显示来源数据源标识

#### 3.3.2 搜索算法要求
- **REQ-SEARCH-ALG-001**: 搜索索引应使用前缀基单词索引以实现 O(1) 查找性能
- **REQ-SEARCH-ALG-002**: 搜索应实现在分隔符和驼峰命名间分割文本的分词
- **REQ-SEARCH-ALG-003**: 索引系统应有效处理复合单词和多词标签
- **REQ-SEARCH-ALG-004**: 搜索算法应通过 350 毫秒防抖提供实时结果以实现流畅 UI 体验

#### 3.3.3 搜索性能
- **REQ-SEARCH-PERF-001**: 搜索操作应在 500 毫秒内完成典型查询
- **REQ-SEARCH-PERF-002**: 搜索索引应高效构建而不阻塞 UI
- **REQ-SEARCH-PERF-003**: 搜索应使用优化的数据结构（前缀树）以快速查找

### 3.4 筛选系统

#### 3.4.1 数据源筛选
- **REQ-FILTER-001**: 应用程序应提供数据源列表，显示所有已加载的JSON文件
- **REQ-FILTER-002**: 用户应能够通过勾选/取消勾选控制每个数据源在时间轴上的显示状态
- **REQ-FILTER-003**: 筛选状态应实时更新视图，无需重启应用
- **REQ-FILTER-004**: 筛选状态应持久化存储，跨会话保持用户选择
- **REQ-FILTER-005**: 应用程序应提供"全选"和"全不选"的快捷操作
- **REQ-FILTER-006**: 筛选界面应显示每个数据源的事件数量和颜色标识

### 3.5 文章系统

#### 3.5.1 内容显示
- **REQ-ARTICLE-001**: 用户应能够查看时间轴事件的详细文章
- **REQ-ARTICLE-002**: 文章应支持 HTML 格式的完整内容展示
- **REQ-ARTICLE-003**: 应用程序应提供返回时间轴的导航
- **REQ-ARTICLE-004**: 文章应显示事件的来源数据源信息

#### 3.5.2 标签与简述气泡
- **REQ-SUMMARY-001**: 时间轴上的每个事件默认显示标签（纯文本）
- **REQ-SUMMARY-002**: 点击标签应弹出简述气泡，显示 Markdown 格式的简述内容
- **REQ-SUMMARY-003**: 简述气泡底部应显示"详情"按钮
- **REQ-SUMMARY-004**: 点击"详情"按钮应跳转到独立页面显示完整描述内容

### 3.6 时间线对比功能

#### 3.6.1 对比视图
- **REQ-COMPARE-001**: 对比视图应支持同时显示 2-4 个数据源的时间线
- **REQ-COMPARE-002**: 对比视图应支持同步滚动（所有时间轴同步移动）
- **REQ-COMPARE-004**: 对比视图应提供时间对齐参考线
- **REQ-COMPARE-005**: 对比视图应支持缩放级别同步

## 4. 技术要求

### 4.1 架构要求

#### 4.1.1 状态管理
- **REQ-ARCH-STATE-001**: 应用程序应使用 Riverpod 进行状态管理
- **REQ-ARCH-STATE-002**: 状态初始化应使用适当的加载状态高效处理
- **REQ-ARCH-STATE-003**: 应使用 Provider 实现依赖注入
- **REQ-ARCH-STATE-004**: 应用程序初始化状态应包括加载、成功和错误状态以及适当的用户反馈
- **REQ-ARCH-STATE-005**: 状态转换应高效处理并配备适当的错误恢复机制

#### 4.1.2 渲染系统
- **REQ-ARCH-RENDER-001**: 应用程序应使用 RenderBox 实现自定义渲染以提高性能
- **REQ-ARCH-RENDER-002**: 动画系统应与渲染管道集成
- **REQ-ARCH-RENDER-003**: 滚动物理应使用平台适当行为实现
- **REQ-ARCH-RENDER-004**: 动画帧应使用 SchedulerBinding 高效调度
- **REQ-ARCH-RENDER-005**: 应用程序应使用视口裁剪仅渲染可见项目加上缓冲区
- **REQ-ARCH-RENDER-006**: 静态元素应使用 PictureLayer 缓存以最少化重绘操作
- **REQ-ARCH-RENDER-007**: 动画重绘应使用 RepaintBoundary 小部件隔离

### 4.2 性能要求

#### 4.2.1 性能标准
- **REQ-PERF-001**: 应用程序在滚动和交互期间应维持 30fps
- **REQ-PERF-002**: 初始加载时间在中端设备上不应超过 5 秒
- **REQ-PERF-003**: 正常操作期间内存使用应保持在 200MB 以下
- **REQ-PERF-004**: 包含数千条目的时间轴应平滑渲染

#### 4.2.2 优化要求
- **REQ-PERF-OPT-001**: 屏幕外项目应从渲染中剔除
- **REQ-PERF-OPT-002**: 滚动期间应最小化布局计算
- **REQ-PERF-OPT-003**: 应用程序应实施基于视口的裁剪以仅渲染可见项目加上缓冲区
- **REQ-PERF-OPT-004**: 静态元素应使用 PictureLayer 缓存以最少化重绘操作
- **REQ-PERF-OPT-005**: 动画重绘应使用 RepaintBoundary 小部件隔离
- **REQ-PERF-OPT-006**: 应用程序应在用户交互后检测稳态并减少渲染频率
- **REQ-PERF-OPT-007**: 应实施画布裁剪以仅渲染时间轴的可见部分
- **REQ-PERF-OPT-008**: 帧调度应优化以防止不必要的动画帧

### 4.3 数据要求

#### 4.3.1 数据结构
- **REQ-DATA-001**: 时间线数据应以 JSON 格式存储以提高灵活性
- **REQ-DATA-002**: 事件数据应包括开始/结束时间、标签、简述（summary）和描述（description）。简述为 Markdown 格式的小段落，在时间轴上以气泡形式显示；描述为 HTML 格式的完整文件，在详情页展示
- **REQ-DATA-003**: 颜色方案应可配置为时间段
- **REQ-DATA-004**: 层次关系（时代包含事件）应受支持
- **REQ-DATA-005**: 时间轴条目应支持父子关系及递归树遍历
- **REQ-DATA-006**: 条目数据应包含渲染状态（位置、透明度、速度）以实现平滑动画
- **REQ-DATA-007**: 颜色管理应支持时间段之间的插值和渐变效果
- **REQ-DATA-008**: JSON 数据格式应支持扩展字段以适应不同主题需求

#### 4.3.2 数据加载
- **REQ-DATA-LOAD-001**: 数据应异步加载而不阻塞 UI
- **REQ-DATA-LOAD-002**: 应为数据加载故障实现错误处理
- **REQ-DATA-LOAD-003**: 数据加载期间应显示进度指示器
- **REQ-DATA-LOAD-004**: 数据解析应妥善处理格式错误的条目
- **REQ-DATA-LOAD-005**: 应支持从本地文件系统和应用内资源加载 JSON
- **REQ-DATA-LOAD-006**: 应支持热重载 JSON 数据以便开发调试

### 4.4 兼容性要求

#### 4.4.1 平台支持
- **REQ-COMPAT-001**: 应用程序应支持最新版本 Flutter（3.x）
- **REQ-COMPAT-002**: 最小 Android SDK 级别应为 21
- **REQ-COMPAT-003**: 最小 iOS 部署目标应为 12.0
- **REQ-COMPAT-004**: 应提供带响应式设计的网页支持

#### 4.4.2 依赖管理
- **REQ-DEP-001**: 依赖应保持更新并修补安全漏洞
- **REQ-DEP-002**: 应谨慎管理破坏性依赖变更

## 9. JSON 数据格式规范

### 9.1 基本结构
```json
{
  "metadata": {
    "name": "数据源名称",
    "version": "1.0",
    "description": "数据源描述",
    "colorScheme": {
      "primary": "#FF5722",
      "secondary": "#FFC107"
    }
  },
  "events": [
    {
      "date": -13800000000,
      "label": "事件标题",
      "summary": "简述内容，Markdown格式的小段落，在时间轴气泡中显示",
      "description": "description_file.html",
      "type": "incident",
      "background": [0, 38, 75],
      "accent": [246, 76, 130, 255]
    }
  ]
}
```

### 9.2 字段说明
- `metadata`: 数据源元信息
  - `name`: 数据源显示名称
  - `version`: 数据格式版本
  - `description`: 数据源描述
  - `colorScheme`: 配色方案
- `events`: 事件数组
  - `date`: 事件时间（负数表示公元前）
  - `label`: 事件标题（纯文本，显示在时间轴上）
  - `summary`: 简述内容，Markdown格式，点击标签后在气泡中显示（可选）
  - `description`: 描述文件名，HTML格式，在详情页展示（可选）
  - `type`: 事件类型（"era" 或 "incident"）
  - `background`: 背景色 RGB 值
  - `accent`: 强调色 RGBA 值
